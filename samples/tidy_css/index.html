Tidy CSS

Пример иллюстрирует подход к оформению таблиц стилей, позволяющих аккуратно разделить DOM страницы и ее отображение.

В идеале, элементы DOM должны отображать исключительно взаимосвязи элементов страницы, а все вопросы ее отображения должны быть вынесены в CSS.

Используемый подход: имеем классы элементов DOM, которые отображают семантику, а в целях отображения дополнительно описываем классы отображения (View Classes), которые отделены от классов DOM, и определяются как множество элементов DOM через набор CSS-селекторов.

Классы отображения определяются на основе схожих элементов дизайна у разных элементов страницы.

На основе определяемых базовых классов можно создавать производные от них путем обобщения (в ООП совсем наоборот - производные классы получаются из базовых путем конкретизации).

Данный пример представляет собой страничку, собранную из двух модулей - предполагается, что они создавались независимо разными разработчиками, и включают в себя базовые таблицы стилей, минимально необходимые для корректного отображения. Главный модуль приложения на основании стилей, полученных от дизайнера, определяет общую таблицу стилей, приводящую все отображение к единому внешнему виду.

В данном примере созданы два вида отображения, реализованных исключительно средствами CSS - простое отображение, и имитация мобильной версии.
